下面是整理后的文档版，核心围绕 Dir 方向抽象 + Tile 面坐标收集 + Up/Down 垂直命中收集。
ThreeTile 方向与邻居收集设计文档
1. 目标
当前需要优化的核心逻辑是：
Tile = 一个原点 + 一个体积
方向 = Tile 某一侧的面
Collect = 遍历这个面的每一个点，然后查找对应方向上的邻居 Tile

目标不是一开始就做过度抽象，而是保留原来清晰的业务结构，同时去掉热路径中的 GC：
不要 List
不要 ToArray
不要 IEnumerable
不要 HashSet
尽量使用 Span / stackalloc / BitSet


2. 核心模型
2.1 Tile 的空间表达
一个 Tile 可以看作：
Position = 原点坐标，packed(x, y, z)
Volume   = 体积，packed(dx, dy, dz)

例如默认麻将体积：
Volume = (2, 2, 1)

表示这个 Tile 占据：
x 方向 2 格
y 方向 2 格
z 方向 1 层


3. Dir：方向枚举
用统一的 Dir 代替原来的多个方向函数。
public enum Dir
{
    Left = 0,
    Right = 1,
    Behind = 2,
    Front = 3,
    Down = 4,
    Up = 5
}

语义：
Left    x - 方向
Right   x + 方向
Behind  y - 方向
Front   y + 方向
Down    z - 方向
Up      z + 方向


4. DirOffset：单个方向偏移
DirOffset 表示一个 signed 三维偏移。
它可以包含负数，但它本身不是 packed position。
public readonly struct DirOffset
{
    public readonly int Dx;
    public readonly int Dy;
    public readonly int Dz;

    public DirOffset(int dx, int dy, int dz)
    {
        Dx = dx;
        Dy = dy;
        Dz = dz;
    }

    public static readonly DirOffset Left = new(-1, 0, 0);
    public static readonly DirOffset Right = new(1, 0, 0);
    public static readonly DirOffset Behind = new(0, -1, 0);
    public static readonly DirOffset Front = new(0, 1, 0);
    public static readonly DirOffset Down = new(0, 0, -1);
    public static readonly DirOffset Up = new(0, 0, 1);
}

注意：
DirOffset 是 readonly struct。
它本身不会产生 GC。

不要把 DirOffset 设计成 ref struct。
因为后续可能需要放入 Span<DirOffset> 或数组。


5. RegionBounds：边界检查
packed position 不能表达非法坐标。
所以在生成新 position 前，必须先检查 x/y/z 是否越界。
public readonly struct RegionBounds
{
    public readonly int MaxCol;
    public readonly int MaxRow;
    public readonly int MaxLayer;

    public RegionBounds(int maxCol, int maxRow, int maxLayer)
    {
        MaxCol = maxCol;
        MaxRow = maxRow;
        MaxLayer = maxLayer;
    }

    public bool IsInside(int x, int y, int z)
    {
        return (uint)x < (uint)MaxCol
            && (uint)y < (uint)MaxRow
            && (uint)z < (uint)MaxLayer;
    }
}

这里使用：
(uint)x < (uint)MaxCol

可以同时处理：
x < 0
x >= MaxCol


6. TileFaceCollector：收集 Tile 某个方向的面
6.1 设计目的
以前每个方向都有单独函数：
GetPositionLeftNeighbourPositions
GetPositionRightNeighbourPositions
GetPositionFrontNeighbourPositions
GetPositionBehindNeighbourPositions
GetPositionUpNeighbourPositions
GetPositionDownNeighbourPositions

这些函数会产生：
List<int>
ToArray()
IEnumerable

现在统一改成：
origin position + volume + dir
    -> 填充某个方向的 face positions 到 Span<int>


6.2 面的数量
对于体积：
(dx, dy, dz)

不同方向的面大小：
Left / Right      = dy * dz
Front / Behind    = dx * dz
Up / Down         = dx * dy

默认体积 (2, 2, 1)：
Left / Right      = 2
Front / Behind    = 2
Up / Down         = 4

所以默认情况下：
Span<int> facePositions = stackalloc int[4];

就足够。
如果未来体积可能变大，需要用：
max(dx * dy, dx * dz, dy * dz)

计算最大面容量。

6.3 TileFaceCollector 代码
using ThreeTile.Core.ExtensionTools;

public static class TileFaceCollector
{
    public static int FillFacePositions(
        int originPosition,
        int volume,
        Dir dir,
        RegionBounds bounds,
        Span<int> destination)
    {
        var (px, py, pz) = originPosition.UnpackXyz();
        var (dx, dy, dz) = volume.UnpackXyz();

        int count = 0;

        switch (dir)
        {
            case Dir.Left:
            {
                int x = px - 1;

                if ((uint)x >= (uint)bounds.MaxCol)
                    return 0;

                for (int z = 0; z < dz; z++)
                for (int y = 0; y < dy; y++)
                {
                    int yy = py + y;
                    int zz = pz + z;

                    if (!bounds.IsInside(x, yy, zz))
                        continue;

                    destination[count++] = (x, yy, zz).PackXyz();
                }

                return count;
            }

            case Dir.Right:
            {
                int x = px + dx;

                if ((uint)x >= (uint)bounds.MaxCol)
                    return 0;

                for (int z = 0; z < dz; z++)
                for (int y = 0; y < dy; y++)
                {
                    int yy = py + y;
                    int zz = pz + z;

                    if (!bounds.IsInside(x, yy, zz))
                        continue;

                    destination[count++] = (x, yy, zz).PackXyz();
                }

                return count;
            }

            case Dir.Behind:
            {
                int y = py - 1;

                if ((uint)y >= (uint)bounds.MaxRow)
                    return 0;

                for (int z = 0; z < dz; z++)
                for (int x = 0; x < dx; x++)
                {
                    int xx = px + x;
                    int zz = pz + z;

                    if (!bounds.IsInside(xx, y, zz))
                        continue;

                    destination[count++] = (xx, y, zz).PackXyz();
                }

                return count;
            }

            case Dir.Front:
            {
                int y = py + dy;

                if ((uint)y >= (uint)bounds.MaxRow)
                    return 0;

                for (int z = 0; z < dz; z++)
                for (int x = 0; x < dx; x++)
                {
                    int xx = px + x;
                    int zz = pz + z;

                    if (!bounds.IsInside(xx, y, zz))
                        continue;

                    destination[count++] = (xx, y, zz).PackXyz();
                }

                return count;
            }

            case Dir.Down:
            {
                int z = pz - 1;

                if ((uint)z >= (uint)bounds.MaxLayer)
                    return 0;

                for (int y = 0; y < dy; y++)
                for (int x = 0; x < dx; x++)
                {
                    int xx = px + x;
                    int yy = py + y;

                    if (!bounds.IsInside(xx, yy, z))
                        continue;

                    destination[count++] = (xx, yy, z).PackXyz();
                }

                return count;
            }

            case Dir.Up:
            {
                int z = pz + dz;

                if ((uint)z >= (uint)bounds.MaxLayer)
                    return 0;

                for (int y = 0; y < dy; y++)
                for (int x = 0; x < dx; x++)
                {
                    int xx = px + x;
                    int yy = py + y;

                    if (!bounds.IsInside(xx, yy, z))
                        continue;

                    destination[count++] = (xx, yy, z).PackXyz();
                }

                return count;
            }

            default:
                throw new ArgumentOutOfRangeException(nameof(dir));
        }
    }

    public static int GetMaxFacePositionCount(int volume)
    {
        var (dx, dy, dz) = volume.UnpackXyz();

        int yz = dy * dz;
        int xz = dx * dz;
        int xy = dx * dy;

        return Math.Max(xy, Math.Max(xz, yz));
    }
}


7. 水平方向邻居收集
对于 Left / Right / Front / Behind，逻辑是：
取 Tile 某个方向的面
遍历面的每一个位置
查这个位置是否有 Tile
如果有，加入结果 bitset

示例：
public bool CollectSideNeighborTiles(
    Tile tile,
    Dir dir,
    Span<ulong> hitBits,
    ReadOnlySpan<ulong> presentBits,
    ReadOnlySpan<ulong> excludeBits = default)
{
    if (tile is null)
        throw new ArgumentNullException(nameof(tile));

    if (dir is Dir.Up or Dir.Down)
        throw new ArgumentException("Side neighbor only supports Left / Right / Front / Behind.", nameof(dir));

    bool changed = false;

    Span<int> facePositions = stackalloc int[4];

    int faceCount = TileFaceCollector.FillFacePositions(
        tile.TilePositionIndex,
        tile.Volume,
        dir,
        Parent.Bounds,
        facePositions);

    for (int i = 0; i < faceCount; i++)
    {
        int pos = facePositions[i];

        int neighborIndex = Parent.GetTileIndexAtPosition(pos);

        if (neighborIndex == TileMappingTable.EmptyTileIndex)
            continue;

        if (neighborIndex == tile.Index)
            continue;

        if (!BitOps.Contains(presentBits, neighborIndex))
            continue;

        if (BitOps.Contains(excludeBits, neighborIndex))
            continue;

        bool already = BitOps.Contains(hitBits, neighborIndex);

        BitOps.Set(hitBits, neighborIndex);

        changed |= !already;
    }

    return changed;
}


8. 垂直方向命中收集
8.1 业务模型
Up / Down 和左右前后不同。
它不是只看紧贴面的那一层，而是：
取 Up / Down 面
for 面的每一个点：
    固定 x/y
    不断改变 z
    找到第一个命中的 Tile

也就是：
Tile
  -> 一个点
  -> 一个体积
  -> 一个方向面
  -> 面上的每个点
  -> 不断改变 z 扫描


8.2 Up / Down 公共函数
因为 Up 和 Down 大部分逻辑相同，所以抽成一个私有函数：
private bool CollectVerticalHitTile(
    Tile tile,
    Dir dir,
    Span<ulong> hitBits,
    ReadOnlySpan<ulong> presentBits,
    ReadOnlySpan<ulong> excludeBits = default)
{
    if (tile is null)
        throw new ArgumentNullException(nameof(tile));

    if (dir != Dir.Up && dir != Dir.Down)
        throw new ArgumentException("Only Up / Down are supported.", nameof(dir));

    int zStep = dir == Dir.Up ? 1 : -1;

    bool changed = false;

    Span<int> facePositions = stackalloc int[4];

    int faceCount = TileFaceCollector.FillFacePositions(
        tile.TilePositionIndex,
        tile.Volume,
        dir,
        Parent.Bounds,
        facePositions);

    for (int i = 0; i < faceCount; i++)
    {
        int facePos = facePositions[i];

        for (int z = facePos.Z();
             dir == Dir.Up ? z < Parent.MaxLayer : z >= 0;
             z += zStep)
        {
            int scanPos = facePos.WithZ(z);

            int neighborIndex = Parent.GetTileIndexAtPosition(scanPos);

            if (neighborIndex == TileMappingTable.EmptyTileIndex)
                continue;

            if (neighborIndex == tile.Index)
                continue;

            if (!BitOps.Contains(presentBits, neighborIndex))
                continue;

            if (BitOps.Contains(excludeBits, neighborIndex))
                continue;

            bool already = BitOps.Contains(hitBits, neighborIndex);

            BitOps.Set(hitBits, neighborIndex);

            changed |= !already;

            break;
        }
    }

    return changed;
}


8.3 Up / Down 入口函数
public bool CollectUpHitTile(
    Tile tile,
    Span<ulong> hitBits,
    ReadOnlySpan<ulong> presentBits,
    ReadOnlySpan<ulong> excludeBits = default)
{
    return CollectVerticalHitTile(
        tile,
        Dir.Up,
        hitBits,
        presentBits,
        excludeBits);
}

public bool CollectDownHitTile(
    Tile tile,
    Span<ulong> hitBits,
    ReadOnlySpan<ulong> presentBits,
    ReadOnlySpan<ulong> excludeBits = default)
{
    return CollectVerticalHitTile(
        tile,
        Dir.Down,
        hitBits,
        presentBits,
        excludeBits);
}


9. HashSet 改为 BitSet
原来：
HashSet<int> hitTileIndexes

现在改成：
Span<ulong> hitBits

原来：
ReadOnlySpan<bool> exclude

现在建议改成：
ReadOnlySpan<ulong> excludeBits

原因：
HashSet<int> 会产生堆对象和哈希操作。
ReadOnlySpan<bool> 虽然不一定 GC，但和 BitSet 体系不统一。
Span<ulong> 可以用位操作表示 tileIndex 集合，更适合热路径。


10. BitOps 要求
BitOps.Contains 必须支持 default span。
[MethodImpl(MethodImplOptions.AggressiveInlining)]
public static bool Contains(ReadOnlySpan<ulong> bits, int bit)
{
    int wordIndex = bit >> 6;

    if ((uint)wordIndex >= (uint)bits.Length)
        return false;

    ulong mask = 1UL << (bit & 63);

    return (bits[wordIndex] & mask) != 0;
}

BitOps.Set：
[MethodImpl(MethodImplOptions.AggressiveInlining)]
public static void Set(Span<ulong> bits, int bit)
{
    int wordIndex = bit >> 6;

#if DEBUG
    if ((uint)wordIndex >= (uint)bits.Length)
        throw new ArgumentOutOfRangeException(nameof(bit));
#endif

    ulong mask = 1UL << (bit & 63);

    bits[wordIndex] |= mask;
}


11. 最终调用示例
Span<ulong> hitBits = stackalloc ulong[3];      // tileIndex <= 160 时通常够用
ReadOnlySpan<ulong> presentBits = state.PresentBits;
ReadOnlySpan<ulong> excludeBits = excludeBuffer;

CollectUpHitTile(
    tile,
    hitBits,
    presentBits,
    excludeBits);

如果 tileIndex 从 1 开始，且最大 tile 数是 160，那么需要：
(160 + 64) / 64 = 3 个 ulong

更通用的计算：
int wordCount = (maxTileIndex + 64) >> 6;

或者：
int wordCount = (maxTileIndex + 63) >> 6;

具体取决于你的 tileIndex 是 0-based 还是 1-based。

13. 最终设计总结
最终模型：
Dir
  统一方向表达。

DirOffset
  表达 signed x/y/z 偏移。
  适合单点移动。

TileFaceCollector
  根据 Tile 的 origin position + volume + dir，收集方向面上的位置。

CollectSideNeighborTiles
  用于 Left / Right / Front / Behind。
  只查当前方向面上的直接邻居。

CollectVerticalHitTile
  用于 Up / Down。
  取 Up / Down 面后，对每个面点固定 x/y，不断改变 z 扫描。

hitBits
  使用 Span<ulong> 表示命中集合。

excludeBits
  使用 ReadOnlySpan<ulong> 表示排除集合。

presentBits
  使用 ReadOnlySpan<ulong> 表示当前仍存在于盘面的 Tile 集合。

一句话：
方向逻辑用 Dir 统一；
空间逻辑用 TileFaceCollector 统一；
收集结果用 Span<ulong> 统一；
Up / Down 用 CollectVerticalHitTile 合并；
水平方向用 CollectSideNeighborTiles 处理。
